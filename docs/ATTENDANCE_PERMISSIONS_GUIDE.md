# 考勤權限管理功能說明

## 功能概述

考勤權限管理功能允許系統管理員設定特定員工對不同部門的考勤審核權限，包括：

- **請假申請審核權限**: 可審核指定部門員工的請假申請
- **加班申請審核權限**: 可審核指定部門員工的加班申請  
- **調班申請審核權限**: 可審核指定部門員工的調班申請
- **班表管理權限**: 可編輯和管理指定部門員工的班表

## 權限範圍

系統支援以下 7 個部門的權限管理：
- 總務
- 倉儲  
- 清潔
- 廚務
- 照服
- 社工
- 護理

## 使用方法

### 1. 訪問權限管理頁面
- 管理員登入系統後，進入「系統設定」
- 在「考勤管理設定」區域找到「考勤權限管理」
- 點擊進入權限管理頁面

### 2. 新增權限設定
1. 點擊「新增權限設定」按鈕
2. 選擇要授予權限的員工（只顯示尚未設定權限的員工）
3. 為每種權限類型勾選對應的部門：
   - 請假審核權限：該員工可審核所選部門的請假申請
   - 加班審核權限：該員工可審核所選部門的加班申請
   - 調班審核權限：該員工可審核所選部門的調班申請
   - 班表管理權限：該員工可管理所選部門的員工班表
4. 至少需要選擇一個權限類型和部門
5. 點擊「新增權限設定」完成設定

### 3. 編輯權限設定
1. 在權限設定列表中找到要編輯的員工
2. 點擊「編輯」圖示
3. 修改權限設定（勾選/取消勾選部門）
4. 點擊「儲存變更」

### 4. 刪除權限設定
1. 在權限設定列表中找到要刪除的員工
2. 點擊「刪除」圖示
3. 確認刪除操作

## 權限效果

設定權限後，被授權的員工將在相應的審核頁面看到：

### 請假管理
- 可在請假管理頁面看到所負責部門的請假申請
- 可批准或拒絕這些申請

### 加班管理  
- 可在加班管理頁面看到所負責部門的加班申請
- 可批准或拒絕這些申請

### 調班管理
- 可在調班管理頁面看到所負責部門的調班申請  
- 可批准或拒絕這些申請

### 班表管理
- 可在班表管理頁面編輯所負責部門員工的班表
- 可新增、修改、刪除班表記錄

## 數據庫結構

### attendance_permissions 表
- `id`: 權限設定ID
- `employee_id`: 員工ID（唯一）
- `permissions`: JSON格式的權限設定
  ```json
  {
    "leaveRequests": ["總務", "倉儲"],
    "overtimeRequests": ["總務"],  
    "shiftExchanges": ["總務", "倉儲", "清潔"],
    "scheduleManagement": ["總務"]
  }
  ```
- `created_at`: 創建時間
- `updated_at`: 更新時間

## 安全考慮

1. **權限驗證**: 只有系統管理員（ADMIN角色）可以設定考勤權限
2. **範圍限制**: 員工只能審核被授權部門的申請
3. **操作記錄**: 所有權限變更都會記錄時間戳
4. **唯一性**: 每個員工只能有一個權限設定記錄

## 注意事項

1. 權限設定立即生效，無需重啟系統
2. 刪除員工帳號時，對應的權限設定會自動刪除
3. 部門名稱變更時需要同步更新權限設定  
4. 建議定期檢視權限設定，移除離職員工的權限
5. 系統管理員永遠擁有所有權限，無需額外設定

## 安裝說明

1. 執行數據庫遷移：
   ```bash
   chmod +x migrate-attendance-permissions.sh
   ./migrate-attendance-permissions.sh
   ```

2. 重新啟動開發服務器：
   ```bash
   npm run dev
   ```

3. 訪問系統設定頁面驗證功能是否正常

## 故障排除

### 問題：無法看到權限管理選項
- 確認當前用戶具有 ADMIN 權限
- 檢查系統設定頁面是否正確載入

### 問題：權限設定無法儲存
- 檢查數據庫連接是否正常
- 確認 Prisma 遷移是否成功執行
- 查看瀏覽器控制台是否有錯誤訊息

### 問題：權限不生效
- 確認權限設定已成功儲存
- 檢查審核頁面的權限檢查邏輯
- 確認員工的部門資訊是否正確
