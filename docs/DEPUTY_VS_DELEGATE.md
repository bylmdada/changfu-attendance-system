# 代理人設定 vs 審核代理人管理

本文件說明系統中兩種代理人設定功能的差異與使用情境。

---

## 概述

系統提供兩種代理人機制：

| 功能 | 代理人設定 | 審核代理人管理 |
|-----|-----------|---------------|
| **入口** | 審核流程管理 → 代理人設定 Tab | 系統設定 → 審核代理人 |
| **對象** | 部門主管專用 | 任意員工可用 |
| **用途** | 組織架構備援 | 臨時權限委託 |

---

## 1. 代理人設定（部門主管代理人）

### 位置
系統設定 → 審核流程管理 → 代理人設定 Tab

### 資料模型
- 資料表：`ManagerDeputy`
- 關聯：`DepartmentManager` → `ManagerDeputy`

### 使用情境
當**部門主管**無法審核時（如請假、出差），由指定的代理人代為審核該部門員工的申請。

### 設定方式
1. 先在「部門主管」Tab 建立部門主管
2. 再為該主管指定代理人
3. 代理人可設定生效期間（可選）

### 結構示意
```
護理部
├── 主管：王主任（ID: 101）
│   ├── 代理人 1：李副主任（ID: 102）
│   │   └── 生效期間：常態代理
│   └── 代理人 2：陳組長（ID: 103）
│       └── 生效期間：2025-01-15 ~ 2025-01-20

日照中心
├── 主管：林主管（ID: 201）
│   └── 代理人：張副主管（ID: 202）
│       └── 生效期間：常態代理
```

### 特點
- ✅ 綁定部門架構
- ✅ 代理範圍：該部門所有員工的申請
- ✅ 適合正式的組織備援機制
- ✅ 可設定多位代理人

---

## 2. 審核代理人管理

### 位置
系統設定 → 審核代理人

### 資料模型
- 資料表：`ApprovalDelegate`
- 欄位：
  - `delegatorId`：委託人
  - `delegateId`：被委託人
  - `startDate`：生效開始日
  - `endDate`：生效結束日
  - `isActive`：是否啟用

### 使用情境
**任何有審核權限的員工**臨時無法審核時，可將權限委託給他人。

### 設定方式
1. 進入審核代理人管理頁面
2. 選擇委託人和被委託人
3. 設定生效期間
4. 啟用代理設定

### 結構示意
```
委託記錄：
┌─────────┬─────────┬────────────────────┬────────┐
│ 委託人   │ 被委託人 │ 生效期間            │ 狀態   │
├─────────┼─────────┼────────────────────┼────────┤
│ 王主任   │ 張副理   │ 2025-01-01 ~ 01-10 │ 啟用中 │
│ 李經理   │ 陳專員   │ 2025-01-05 ~ 01-08 │ 啟用中 │
│ 林主管   │ 黃組長   │ 2024-12-01 ~ 12-15 │ 已過期 │
└─────────┴─────────┴────────────────────┴────────┘
```

### 特點
- ✅ 不限定部門
- ✅ 適合臨時性的權限委託
- ✅ 明確的生效期間
- ✅ 可隨時啟用/停用

---

## 功能對比表

| 比較項目 | 代理人設定 | 審核代理人管理 |
|---------|-----------|---------------|
| **設定對象** | 僅限部門主管 | 任意員工 |
| **代理範圍** | 該部門所有申請 | 委託人的所有審核權限 |
| **與部門關聯** | 是 | 否 |
| **生效期間** | 可選（常態或期間） | 必填 |
| **主要用途** | 組織架構正式備援 | 臨時性權限委託 |
| **資料表** | `ManagerDeputy` | `ApprovalDelegate` |
| **管理入口** | 審核流程管理頁面內 | 獨立管理頁面 |

---

## 使用建議

### 場景 1：部門主管請長假
**建議使用**：代理人設定

**原因**：主管的代理人是正式的組織備援，應在部門主管設定中明確指定。

### 場景 2：臨時出差 3 天
**建議使用**：審核代理人管理

**原因**：短期、明確期間的委託，使用審核代理人管理更為靈活。

### 場景 3：主管有常態副手
**建議使用**：代理人設定（常態代理）

**原因**：副手作為正式的組織備援，應在代理人設定中設為「常態代理」。

---

## 注意事項

1. **權限繼承**：代理人會繼承委託人的審核權限，請謹慎指定。

2. **期間設定**：
   - 代理人設定：若未設定期間，視為「常態代理」
   - 審核代理人管理：必須設定明確期間

3. **優先順序**：當同時存在多個代理設定時，系統會依據生效期間判斷。

4. **審計追蹤**：所有代理審核操作會記錄代理人資訊，可供稽核查詢。

## 相關資料表結構

### ManagerDeputy（部門主管代理人）
```prisma
model ManagerDeputy {
  id          Int       @id @default(autoincrement())
  managerId   Int       // 部門主管 ID
  employeeId  Int       // 代理人員工 ID
  startDate   DateTime? // 開始日期（空值表示常態代理）
  endDate     DateTime? // 結束日期
  isActive    Boolean   @default(true)
  createdAt   DateTime  @default(now())

  manager     DepartmentManager @relation(...)
  employee    Employee          @relation("DeputyEmployee", ...)
}
```

### ApprovalDelegate（審核代理人）
```prisma
model ApprovalDelegate {
  id          Int       @id @default(autoincrement())
  delegatorId Int       // 委託人員工 ID
  delegateId  Int       // 被委託人員工 ID
  startDate   DateTime  // 開始日期
  endDate     DateTime  // 結束日期
  reason      String?   // 委託原因
  isActive    Boolean   @default(true)
  createdAt   DateTime  @default(now())

  delegator   Employee  @relation("Delegator", ...)
  delegate    Employee  @relation("Delegate", ...)
}
```
