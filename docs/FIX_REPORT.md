## 套用週模版問題修正報告

### 🔍 問題診斷
1. **模版格式不匹配**: API 預期 `day1-day7` 但實際檔案使用 `monday-sunday`
2. **員工資料獲取**: 內部 HTTP 請求可能失敗
3. **類型轉換**: templateId 可能需要確保數字類型比較
4. **錯誤處理**: 需要更詳細的日誌來診斷問題

### ✅ 已完成修正
1. **更新 Template 介面**: 改為使用 `monday-sunday` 格式
2. **修正日期映射邏輯**: 使用 `dayNames` 陣列正確映射星期
3. **簡化員工資料**: 暫時使用模擬資料避免內部請求問題
4. **增強錯誤處理**: 添加詳細的 console.log 和錯誤資訊
5. **添加 OPTIONS 方法**: 處理 CORS 預檢請求
6. **確保數字比較**: 使用 Number() 確保 templateId 正確比較

### 📝 修正後的 API 流程
```
1. 接收請求參數 (templateId: 3, year: 2025, month: 7)
2. 載入週模版檔案 (/data/weekly-templates.json)
3. 尋找 ID=3 的模版 ("112" 模版)
4. 載入現有排程 (/data/schedules.json)
5. 獲取員工列表 (使用模擬資料)
6. 生成 2025年7月的所有日期
7. 移除該月現有排程
8. 根據模版為每個員工生成新排程
9. 儲存合併後的排程
10. 回傳成功訊息
```
