# GPS打卡設定系統功能說明

## 🎯 系統概述
GPS打卡設定系統是一個完整的位置管理解決方案，讓管理員可以靈活配置GPS打卡功能，確保員工在指定位置範圍內進行打卡。

## 🔧 主要功能

### 1. GPS基本設定
**位置：** `/system-settings/gps-attendance`

#### 功能選項：
- **啟用GPS打卡** 
  - 開關GPS定位驗證功能
  - 關閉時員工可在任何位置打卡

- **GPS精確度要求**
  - 設定範圍：10-500公尺
  - 控制GPS定位精確度要求
  - 數值越小要求越嚴格

- **允許離線模式**
  - 無GPS訊號時是否允許打卡
  - 適用於地下室或訊號不佳環境

- **離線寬限時間**
  - 設定範圍：1-60分鐘
  - 離線模式下的最長等待時間

- **最大距離偏差**
  - 設定範圍：5-100公尺
  - 允許的GPS座標偏差範圍

- **驗證超時時間**
  - 設定範圍：10-120秒
  - GPS位置驗證的最長等待時間

- **啟用位置歷史記錄**
  - 保存員工打卡位置歷史
  - 用於統計分析和追蹤

- **需要地址資訊**
  - 打卡時是否須提供詳細地址
  - 提供更完整的位置資訊

### 2. 允許位置管理
**位置：** `/system-settings/gps-attendance` (下半部分)

#### 功能特點：
- **新增位置**
  - 手動輸入GPS座標
  - 使用當前位置自動獲取座標
  - 設定允許範圍半徑

- **位置資訊管理**
  - 位置名稱設定
  - 適用部門限制
  - 工作時間設定
  - 啟用/停用狀態

- **位置操作**
  - 編輯現有位置
  - 刪除不需要的位置
  - 批量管理功能

### 3. 打卡驗證流程
**位置：** `/attendance` (打卡頁面)

#### 驗證步驟：
1. **獲取GPS設定**
   - 自動讀取系統設定
   - 動態調整驗證規則

2. **位置檢查**
   - GPS精確度驗證
   - 允許範圍距離計算
   - 地址資訊獲取

3. **錯誤處理**
   - 明確的錯誤訊息
   - 操作指引提示
   - 重試機制

## 🛠️ 技術特點

### 前端功能
- **即時位置狀態顯示**
  - 檢查中、通過、失敗、錯誤狀態
  - 視覺化狀態指示器
  - 詳細錯誤說明

- **智能表單驗證**
  - 即時輸入檢查
  - 範圍限制控制
  - 依設定動態調整

### 後端API
- **設定管理API** (`/api/system-settings/gps-attendance`)
  - GET: 獲取設定
  - POST: 更新設定
  - PUT: 重置設定
  - PATCH: 部分更新

- **位置管理API** (`/api/attendance/allowed-locations`)
  - CRUD完整操作
  - 位置驗證邏輯
  - 距離計算功能

- **打卡驗證API** (`/api/attendance/verify-clock`)
  - 整合GPS設定檢查
  - 位置資料儲存
  - 錯誤處理機制

### 資料庫擴展
- **AttendanceRecord模型擴展**
  ```sql
  -- 新增GPS相關欄位
  clockInLatitude    FLOAT
  clockInLongitude   FLOAT  
  clockInAccuracy    FLOAT
  clockInAddress     TEXT
  clockOutLatitude   FLOAT
  clockOutLongitude  FLOAT
  clockOutAccuracy   FLOAT
  clockOutAddress    TEXT
  ```

## 📱 使用流程

### 管理員設定流程
1. **進入系統設定**
   - Dashboard → 系統設定 → GPS打卡設定

2. **配置基本設定**
   - 開啟GPS功能
   - 設定精確度要求
   - 配置離線模式

3. **管理允許位置**
   - 新增公司/分店位置
   - 設定允許範圍
   - 配置部門限制

4. **儲存並測試**
   - 儲存設定
   - 測試打卡功能

### 員工打卡流程
1. **進入打卡頁面**
   - Dashboard → 出勤打卡

2. **位置驗證**
   - 系統自動檢查GPS設定
   - 獲取當前位置
   - 驗證是否在允許範圍

3. **執行打卡**
   - 輸入帳號密碼
   - 系統驗證位置
   - 完成打卡記錄

## 🔐 安全考量

### 隱私保護
- 僅在打卡時獲取位置
- 位置資料僅用於考勤驗證
- 可選擇是否保存歷史記錄

### 錯誤容忍
- 離線模式支援
- GPS精確度容差
- 多重驗證機制

### 權限控制
- 僅管理員可修改設定
- 員工僅可查看打卡狀態
- API權限驗證
