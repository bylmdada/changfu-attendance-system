## 系統概述

本系統透過 GPS 定位驗證員工打卡位置，確保員工在指定的工作地點範圍內打卡。系統支援兩種打卡入口：

| 入口 | 路徑 | 說明 |
|------|------|------|
| **快速打卡** | 登入頁面 → 快速打卡分頁 | 輸入員編與密碼直接打卡，不需登入系統 |
| **系統打卡** | 登入系統 → 打卡管理 | 登入後在打卡管理頁面打卡 |

兩種入口均使用相同的 GPS 驗證邏輯與 API。

## GPS 設定參數

管理員可在「系統設定 → GPS 打卡設定」頁面調整以下參數：

| 參數 | 說明 | 預設值 |
|------|------|--------|
| **GPS 啟用** (`enabled`) | 是否開啟 GPS 定位驗證 | `true` |
| **精確度要求** (`requiredAccuracy`) | GPS 精確度門檻（公尺），超過此值將拒絕打卡 | `100m` |
| **離線模式** (`allowOfflineMode`) | 是否允許無法取得 GPS 時仍可打卡 | `false` |
| **離線寬限時間** (`offlineGracePeriod`) | 離線模式下的寬限時間（分鐘） | `5` |
| **最大距離偏差** (`maxDistanceVariance`) | 允許的最大位置偏差（公尺） | `20m` |
| **驗證超時** (`verificationTimeout`) | GPS 定位等待時間上限（秒） | `30` |
| **位置歷史記錄** (`enableLocationHistory`) | 是否記錄每次打卡的 GPS 座標 | `true` |
| **地址資訊要求** (`requireAddressInfo`) | 是否要求取得地址文字資訊 | `false` |

## 允許打卡地點

系統可設定多個允許的打卡地點，每個地點包含：

| 欄位 | 說明 |
|------|------|
| **名稱** | 地點名稱（如「溪北輔具中心」） |
| **經緯度** | 地點中心的 GPS 座標 |
| **半徑** | 允許打卡的範圍半徑（公尺），員工需在此圓形範圍內 |
| **所屬部門** | 可限定特定部門才能在此地點打卡 |
| **啟用狀態** | 可個別啟用或停用地點 |

### 位置判斷邏輯

系統使用 Haversine 公式計算員工目前位置與各允許地點中心的距離。若距離小於該地點設定的半徑，即判定為「在範圍內」，允許打卡。

## 打卡流程

```
員工按下打卡按鈕
    │
    ▼
GPS 啟用？ ── 否 ──→ 直接打卡（不驗證位置）
    │ 是
    ▼
取得 GPS 座標（高精確度模式）
    │
    ▼
GPS 精確度 ≤ 要求？ ── 否 ──→ 顯示「GPS 精確度不足」錯誤
    │ 是
    ▼
在允許範圍內？ ── 否 ──→ 顯示「不在允許打卡範圍」+ 距離提示
    │ 是
    ▼
WiFi 驗證需要？ ── 是 ──→ 選擇 WiFi SSID → 驗證
    │ 否
    ▼
身分驗證（密碼 / Face ID / 指紋）
    │
    ▼
送出打卡請求（含 GPS 座標）
    │
    ▼
伺服器驗證 + 防偽偵測
    │
    ▼
打卡成功，記錄 GPS 資料
```

### 記錄的 GPS 欄位

每次打卡會記錄以下資料：

| 上班打卡 | 下班打卡 |
|----------|----------|
| `clockInLatitude` | `clockOutLatitude` |
| `clockInLongitude` | `clockOutLongitude` |
| `clockInAccuracy` | `clockOutAccuracy` |
| `clockInAddress` | `clockOutAddress` |

## WiFi 輔助驗證

部分地點可啟用 WiFi SSID 輔助驗證，作為 GPS 之外的第二層位置確認。

| 設定 | 說明 |
|------|------|
| **WiFi 啟用** (`wifiEnabled`) | 該地點是否啟用 WiFi 驗證 |
| **WiFi SSID 清單** (`wifiSsidList`) | 允許的 WiFi 名稱（每行一個） |
| **僅限 WiFi** (`wifiOnly`) | 是否只用 WiFi 驗證（不需 GPS 在範圍內） |

啟用後，員工打卡時需從下拉選單中選擇目前連接的 WiFi 網路名稱，系統會比對是否為該地點的允許清單。

## GPS 防偽偵測

系統內建多層 GPS 欺騙偵測機制，每項檢查會產生風險分數，總分達 **30 分以上** 即標記為可疑：

| 偵測項目 | 說明 | 風險分數 |
|----------|------|----------|
| **座標精度不足** | 緯經度小數位數 < 4 位 | +25 |
| **完美整數座標** | 座標為整數（極不自然） | +40 |
| **異常完美精度** | GPS 精度 = 0 或 < 3 公尺 | +20 |
| **座標跳變** | 60 秒內移動 > 500 公尺 | +35 |
| **瞬間移動** | 5 分鐘內移動 > 10 公里 | +50 |
| **深夜打卡** | 00:00 ~ 05:00 時段打卡 | +15 |
| **頻繁嘗試** | 1 分鐘內 ≥ 3 次打卡嘗試 | +20 |

偵測結果會記錄在稽核日誌中，供管理員事後審查。

## 常見問題排解

### 員工無法打卡：「GPS 精確度不足」

**原因：** 手機 GPS 訊號不佳，精確度超過設定門檻（目前 100m）。

**解決方式：**
1. 移到戶外或窗邊，等待 GPS 訊號穩定
2. 確認手機已開啟「位置服務」（定位功能）
3. 關閉省電模式（可能限制 GPS 精度）
4. 管理員可適當調高 `requiredAccuracy` 值（如 150m 或 200m）

### 員工無法打卡：「不在允許的打卡範圍內」

**原因：** 員工位置超出所有允許地點的半徑範圍。

**解決方式：**
1. 確認員工是否在正確的工作地點
2. 系統會顯示距離最近地點的距離，可據此判斷
3. 管理員可調大該地點的半徑設定
4. 若有新的工作地點，管理員需新增允許地點

### 員工無法打卡：「GPS 定位權限被拒絕」

**原因：** 瀏覽器的定位權限被關閉。

**解決方式：**
1. iOS Safari：設定 → Safari → 位置 → 允許
2. Android Chrome：點擊網址列左側鎖頭圖示 → 位置 → 允許
3. 清除瀏覽器快取後重新開啟網頁

### 員工無法打卡：「GPS 定位超時」

**原因：** 在 15 秒內未能取得 GPS 座標。

**解決方式：**
1. 確認手機 GPS 已開啟
2. 移到訊號較好的位置（避免室內深處）
3. 重新整理頁面後再試

## 管理員操作指引

### 新增允許打卡地點

1. 進入「系統設定 → GPS 打卡設定」
2. 在「允許打卡地點」區塊點擊「新增地點」
3. 填入地點名稱、GPS 座標（緯度、經度）、半徑
4. 選擇所屬部門（可選）
5. 儲存

> 取得 GPS 座標的方法：在 Google Maps 上對目標位置點右鍵，即可複製經緯度。

### 調整 GPS 設定

1. 進入「系統設定 → GPS 打卡設定」
2. 修改相關參數
3. 點擊「儲存設定」

### 查看打卡 GPS 記錄

每筆打卡記錄都包含 GPS 座標和精確度，可在「考勤紀錄」中查看，用於稽核和問題追蹤。
