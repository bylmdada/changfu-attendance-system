# 時薪與加班費計算說明

本文件說明系統中時薪儲存格式、加班費計算邏輯，以及取整策略。

---

## 時薪資料型態

### 為什麼使用浮點數（Float）？

時薪欄位在資料庫中使用 `Float` 型態，而非整數。

**計算公式**：
```
時薪 = 月薪 ÷ 240
```

**範例**：

| 月薪 | 時薪（精確值） | 若取整數 | 誤差 |
|-----|--------------|---------|------|
| 30,000 | 125.00 | 125 | 0 |
| 32,000 | 133.33... | 133 | -0.33 |
| 35,000 | 145.83... | 146 | +0.17 |
| 45,000 | 187.50 | 188 | +0.50 |

**結論**：使用浮點數可保留計算精確度。

---

## 時薪 × 240 = 月薪（整數）

### 數學原理

```
時薪 × 240 = (月薪 ÷ 240) × 240 = 月薪
```

即使時薪是浮點數，乘回 240 後仍會回到原始的月薪整數。

### 浮點數精度處理

電腦浮點數運算可能產生微小誤差：
```javascript
133.33333333333334 * 240 = 32000.000000000004
```

**解決方法**：顯示金額時使用 `Math.round()` 取整

```typescript
const monthlySalary = Math.round(hourlyRate * 240);  // 32000
```

## 加班費計算策略

### 原則：計算過程保持浮點數，最終金額取整

#### ❌ 不建議：各段分別取整

```javascript
const first2Hours = Math.round(hourlyRate * 2 * 1.34);  // 338
const remainingHours = Math.round(hourlyRate * 1 * 1.67); // 210
return first2Hours + remainingHours;  // 548
```

**問題**：每段取整會累積誤差

#### ✅ 建議：最終一次取整（目前採用）

```javascript
const first2Hours = hourlyRate * 2 * 1.34;  // 337.68
const remainingHours = hourlyRate * 1 * 1.67; // 210.41
return Math.round(first2Hours + remainingHours);  // 548
```

**優點**：精確度高，誤差最小

## 加班費計算範例

**假設條件**：
- 月薪 30,000 → 時薪 125
- 平日加班 3 小時

**計算過程**：
```
前2小時：125 × 2 × 1.34 = 335.00
後1小時：125 × 1 × 1.67 = 208.75
合計：335.00 + 208.75 = 543.75
取整：Math.round(543.75) = 544 元
```

## 總結

| 階段 | 是否取整 | 原因 |
|-----|---------|------|
| 時薪儲存 | ❌ Float | 保持精確度 |
| 計算過程 | ❌ 不取整 | 避免累積誤差 |
| 最終加班費 | ✅ Math.round() | 實際發放金額 |
| 顯示月薪 | ✅ Math.round() | 處理浮點誤差 |
