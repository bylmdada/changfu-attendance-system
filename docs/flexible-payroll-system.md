# 靈活薪資條系統設計

## 概述

為了應對未來薪資加薪及保險費調整的需求，我們實現了一個靈活的薪資條系統，該系統支持動態配置薪資項目，並能輕鬆適應各種薪資結構的變化。

## 系統架構

### 1. 數據庫設計

#### PayrollItemConfig 表
- **用途**: 定義所有可能的薪資項目配置
- **關鍵欄位**:
  - `code`: 項目代碼 (如 'BASE_SALARY', 'LABOR_INSURANCE')
  - `name`: 項目名稱 (如 '基本薪資', '勞工保險')
  - `type`: 類型 ('EARNING' 或 'DEDUCTION')
  - `category`: 分類 ('SALARY', 'INSURANCE', 'TAX', 'ALLOWANCE', 'PENSION', 'DEDUCTION')
  - `sortOrder`: 顯示排序順序
  - `isActive`: 是否啟用

#### PayrollItem 表
- **用途**: 記錄每筆薪資記錄的具體項目
- **關鍵欄位**:
  - `payrollId`: 關聯的薪資記錄ID
  - `itemConfigId`: 關聯的項目配置ID
  - `amount`: 金額
  - `quantity`: 數量 (如工時數量)
  - `unitPrice`: 單價
  - `description`: 描述

### 2. API 設計

#### /api/payroll/config
- **GET**: 獲取所有薪資項目配置
- **POST**: 創建新的薪資項目配置

#### /api/payroll/payslip (更新)
- **GET**: 生成包含動態項目的薪資條數據
- 返回結構包含:
  - `earnings`: 收入項目數組
  - `deductions`: 扣除項目數組
  - `summary`: 總計信息

### 3. 前端組件

#### 薪資項目配置頁面 (/payroll/config)
- 管理所有薪資項目配置
- 支持新增、編輯、刪除配置
- 提供分類和排序功能

#### 更新後的薪資條HTML生成
- 動態生成收入和扣除項目
- 支持項目描述和數量顯示
- 保持響應式設計

## 優勢特點

### 1. 高度靈活性
- **動態項目**: 可以輕鬆添加新的薪資項目，如獎金、津貼等
- **配置驅動**: 通過配置表控制項目顯示，而非硬編碼
- **分類管理**: 支持不同類型的項目分類和排序

### 2. 易於維護
- **集中配置**: 所有項目定義集中管理
- **版本控制**: 配置變更可追蹤
- **向後兼容**: 舊數據可通過遷移腳本轉換

### 3. 適應未來變化
- **薪資調整**: 輕鬆適應基本薪資、加班費率變化
- **保險費調整**: 動態調整勞保、健保費率
- **法規變化**: 快速響應稅務和保險政策變化

## 使用場景

### 1. 薪資結構調整
```typescript
// 新增項目配置
{
  code: 'PERFORMANCE_BONUS',
  name: '績效獎金',
  type: 'EARNING',
  category: 'ALLOWANCE',
  sortOrder: 5
}
```

### 2. 保險費率調整
```typescript
// 更新項目金額
{
  code: 'LABOR_INSURANCE',
  amount: newLaborInsuranceAmount,
  description: '更新後的勞保費率'
}
```

### 3. 臨時項目
```typescript
// 添加一次性項目
{
  code: 'YEAR_END_BONUS',
  name: '年終獎金',
  type: 'EARNING',
  category: 'ALLOWANCE',
  amount: bonusAmount
}
```
